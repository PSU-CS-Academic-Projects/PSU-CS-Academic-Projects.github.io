# Contributing to PSU-CS Academic Projects

Thank you for contributing to PSU-CS-Academic-Projects! This organization serves as the central showcase for computer science coursework and personal projects from Palawan State University students.

Before submitting a project, please ensure that your repository follows the organization standards so it is properly formatted and automatically displayed on our [Showcase Website](https://psu-cs-academic-projects.github.io).

## 1. Repository Naming
Create your project repository directly within the **PSU-CS-Academic-Projects** GitHub Organization.
Use clear and descriptive project names.

**Examples of Good Names:**
* `Student-Attendance-System`
* `Library-Management-System`
* `Campus-Navigation-App`
* `Online-Enrollment-System`

**Avoid Generic Names such as:**
* `project`
* `finalproject`
* `thesis`
* `system`
* `group1`

*Note: Ensure your repository visibility is set to **Public**. Private repositories will not be indexed.*

## 2. GitHub Topics (Tags)
Our website relies on GitHub Repository Topics to categorize and filter projects. 
On the right-hand side of your repository's main page, click the gear icon ⚙️ next to "About" to add topics. You must include topics from these three areas:

### Subject (Required)
* `wst` (Web System and Technology)
* `sia` (System Integration Architecture)

### Category
* `web-development`, `mobile-development`, `desktop-application`, `ai`, `machine-learning`, `cybersecurity`, `game-development`, `iot`, `research-project`

### Technologies (Examples)
* `php`, `mysql`, `laravel`, `react`, `nextjs`, `flutter`, `firebase`, `java`, `python`, `unity`, `godot`

## 3. README.md Requirements
Each repository must include a completed `README.md` file.

**Submission Requirements:**
* Project description
* Feature list
* Installation guide
* Screenshots
* Team member information
* Academic year
* Subject information

**Strongly Recommended:**
* Live demo link
* Documentation
* Video demonstration
* Presentation slides
* Future improvements

### Auto-Parser Formatting
The showcase website automatically reads your `README.md` to extract information about your team, video demos, and screenshots. **You must use exact headings for the parser to find them.**

Copy and paste the templates below into your README:

#### Team Members
List all contributing students as bullet points under a `### Members` or `### Contributors` heading.
```markdown
### Members
* John Doe
* Jane Doe
```

#### Video Demonstration
Provide a direct link to your live demo or video walkthrough under a `### Video Demonstration` heading.
```markdown
### Video Demonstration
https://youtube.com/watch?v=your-video-link
```

#### Screenshots
Include images of your project under a `### Screenshots` heading. You can use relative paths or absolute external links (like Imgur).
```markdown
### Screenshots
![Login Page](screenshots/login.png)
![Dashboard](screenshots/dashboard.png)
```

## 4. Updates & Caching
The showcase website uses a static data system. Once your repository is public and properly formatted, it will automatically appear on the website the next time the organization's automation script is run by an administrator.

For complete requirements, please review `PROJECT_SUBMISSION_GUIDE.md` (if available).
